# Market Research Workflow

## Research Design

Start with a narrow question:

- Which user problem is being tested?
- Which audience segment is relevant?
- What decision will the research change?
- What minimum evidence is enough to act?

## Candidate Data Sources

- App store reviews for route planners, hiking apps, GPX tools, offline maps, and weather-planning apps.
- Public competitor websites and help docs.
- Search keyword tools or exported keyword datasets when available.
- Public forums, trail communities, and issue trackers when collection is allowed.
- First-party user feedback supplied by the project owner.

## Analysis Methods

- Tag feedback into jobs-to-be-done, pain points, missing features, trust/safety concerns, and willingness-to-pay signals.
- Segment by activity: hiking, trail running, cycling, driving, travel planning, offline navigation.
- Quantify frequency, severity, and recency where data volume allows.
- Compare Mapping Elf capabilities against competitor promises and user complaints.

## Output Standard

End with evidence-backed insights and implementation handoff:

- Insight
- Evidence
- Confidence
- Product implication
- Suggested owner skill for follow-up
